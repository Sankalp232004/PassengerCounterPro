This repository contains a polished single-file web app that transforms the original passenger counter into a presentable basketball scorecard. It is designed to be compact, easy to demo, and impressive on GitHub for interviews.

How to run

1. Open `index.html` in a browser (double-click or serve with a static server).

Features

- Two-team scoreboard with editable team names
- +3, +2, FT buttons for each team
- Undo stack for plays
- Play history list
- Quarter counter and 10:00 default quarter clock
- Start/Pause timer, Reset clock, Next quarter
- Export play history as CSV
- Theme toggle for light/dark preview
- Local storage persistence so the board survives refreshes

Usage notes

- The app is client-side only and requires no build step. Commit this repo to your GitHub and enable GitHub Pages to host it for a live demo.
- For interviews, present the scoreboard, walk through the features, and show the CSV export and history persistence.
